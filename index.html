    • Field Inspector

      **Please note that this project is still under construction.

      The idea for this app came about during my last job as a civil engineer. I had to perform numerous job site inspections and writing repetitive reports became quite a chore.

      The final version of the app will allow users to conduct construction site inspections using their mobile device. The app will provide checklists and input forms for photo documentation (stored on Amazon S3). Once the user has completed the inspection form and all the photos are uploaded, it will generate a pdf version of the form ready to be emailed to a client or the user himself. The app also automatically scrapes National Oceanic and Atmospheric Administratin (NOAA)'s website using Nokogiri for latest forecast based on the device's GPS coordinates. An email notification can be set up to remind the user that an inspection is required should the forecast indicate more than 50% chance of rain.

      I've written my own user authentication for this app. Ruby on Rails and Amazon S3 handles the backend file and photo storage. I've played with foreman to run background jobs in a single command.

      The biggest challenge I have with this project is converting a form to pdf without using adobe professional. Sure there is a Prawn gem that does that for me, but does it makes sense to store all the input in the backend when the final product is going to be sent as a pdf via email? There are lots of moving parts in this project that adds to the complexity, needles to say, there is never a dull moment working on this.
